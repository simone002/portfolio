Handwriting Classification Framework

This thesis project presents a comprehensive machine learning framework for forensic handwriting analysis to identify authors through character-level analysis. The system combines deep learning techniques with custom preprocessing pipelines to achieve high accuracy in distinguishing handwriting styles of different individuals.

Dataset Creation & Preprocessing

  • CVL Database Integration: Utilized the CVL public database with 310 writers across 7 different texts (German and English)
  • Custom Character Extraction: Developed GUI-based tool for extracting individual characters from word images
  • Advanced Preprocessing Pipeline: Grayscale conversion, adaptive thresholding, noise reduction with cv2.fastNlMeansDenoising, and morphological operations
  • Contour Detection: Automated character segmentation with bounding box calculation and manual correction capabilities
  • 26-Class Dataset: Organized extracted characters into alphabetical classes (a-z) for classification training

CNN Architecture & Fine-tuning

  • Transfer Learning: Fine-tuned ResNet-18, AlexNet, and SqueezeNet models pre-trained on ImageNet
  • Data Augmentation: Brightness adjustment, random rotation, and resizing transformations
  • SGD Optimization: Systematic hyperparameter tuning with learning rate, momentum, and weight decay optimization
  • Regularization: CrossEntropyLoss function with early stopping mechanism to prevent overfitting

Siamese Network Implementation

  • Twin Architecture: Identical CNN subnetworks with shared weights for similarity comparison
  • Contrastive Loss: Custom loss function L(y,D) = (1-y)½D² + y½max(0,m-D)² for distance learning
  • Embedding Network: ResNet18-based feature extraction producing 128-dimensional embeddings
  • Pair Generation: Automated creation of similar/dissimilar character pairs for training
  • Distance Calculation: Multi-level averaging system for comprehensive writer comparison

Siamese Network Performance

  • Training Convergence: Final training loss of 0.0092 with stable convergence after 6000-8000 steps
  • Validation Performance: 0.1011 validation loss with dropout regularization improvements
  • Writer Discrimination: Successfully distinguishes between different writers with distance values ranging 0.95-1.50
  • Multi-subject Validation: Tested on 10 subjects with distance matrix visualization confirming writer separation

Technical Innovation

The framework introduces a novel approach to forensic handwriting analysis by combining character-level CNN classification with Siamese network-based writer verification. The system performs "1-to-n" comparisons for each letter, averaging distances across all characters to produce a comprehensive writer similarity score.

Technologies: Python, PyTorch, PyTorch Lightning, OpenCV, PIL, scikit-learn, cv2, CUDA, TensorFlow

Automatic Pollen Classification System

This project develops an automatic classification system for images of airborne particles acquired via microscope, focusing on distinguishing between different types of pollen and debris particles. The system addresses challenges such as high visual similarity between classes and image variability using Deep Learning architectures, specifically Convolutional Neural Networks (CNNs).

Dataset & Classes

The dataset includes 4 classes: Corylus avellana (normal and anomalous hazelnut pollen), Alnus (alder pollen), and Debris (non-pollen particles). Each sample includes original images, binary masks, and detailed segmentations.

Key Features

  • Advanced Preprocessing: Mask application to isolate particles and remove background noise
  • Multiple CNN Architectures: Comparison of ResNet-34, VGG, and AlexNet models
  • Data Augmentation: Random rotations, horizontal flips, and color jittering
  • Class Imbalance Handling: Weighted random sampling strategy
  • High Performance: All models achieved >96% accuracy on test set

Workflow: Cloud-Native Todo List on AWS

A full-stack web application for task management, fully deployed on AWS with modern architecture and DevOps automation.

Main Features

  • Secure Authentication: Managed with AWS Cognito using JWT tokens.
  • Task Management (CRUD): Create, view, update, and delete tasks with authorization.
  • Protected APIs: Endpoints require valid tokens for access.
  • Event-Driven Notifications: SNS and SQS for asynchronous, decoupled messaging.

Cloud Architecture

  • Frontend: React app hosted as a static website on Amazon S3.
  • Backend: Node.js REST API containerized and running on AWS ECS Fargate, images stored in ECR.
  • Database: MySQL hosted on Amazon RDS.
  • Authentication: AWS Cognito.
  • Logging: CloudWatch Logs.
  • Permissions: IAM role with least privilege assigned to ECS tasks.
